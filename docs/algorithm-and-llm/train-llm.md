# Planner 大模型能力要求与专用训练方案

<!-- SID:algollm.llm.training_plan -->

## 1. 训练目标（面向“准确生成工作流”）

训练目标不是做通用聊天模型，而是训练一个 **Planner 专用模型**，在科研工业设计验证环境中稳定产出高质量候选工作流。

核心目标：

- 准确：满足 ToolKG 与任务约束，生成可执行结构；
- 稳定：同类输入输出波动可控；
- 可恢复：失败上下文下优先给出最小修复；
- 可审计：输出字段支持 HITL 与日志追踪。

______________________________________________________________________

## 2. 能力要求（与系统契约一一对应）

模型必须具备六类能力：

- 约束满足能力：工具可用性、I/O 闭包、参数合法性；
- 结构生成能力：稳定生成 `Plan/PlanPatch/Replan`；
- 候选决策能力：输出 Top-K 与默认建议；
- 风险成本建模能力：给出 `score_breakdown/risk/cost`；
- 修复规划能力：遵循“先 patch 后 replan”；
- 一致性能力：同条件下规划行为可复现。

______________________________________________________________________

## 3. 训练范围与边界

建议路线：

- 模型规模：7B-14B；
- 训练方式：SFT/QLoRA 为主，偏好优化为辅；
- 架构策略：自研模型默认 + 外部强模型回退。

明确边界：

- 不从零预训练通用模型；
- 不改变 FSM/HITL/Agent 职责边界；
- 不移除规则校验器与结构化门禁。

## 4. 数据工程（训练样本的标准来源）

### 4.1 数据来源

从运行系统日志回流构建数据集：

- Task 输入与约束；
- ToolKG 快照与工具选择上下文；
- Planner 候选与最终选择；
- StepResult/FailureCode；
- PendingAction/Decision/EventLog；
- TaskSnapshot（含已成功前缀）。

### 4.2 样本结构（建议）

每条样本建议包含：

- `context`：任务、约束、阶段、前缀进度；
- `candidates`：Top-K 候选及字段完整内容；
- `selected`：系统或人工最终选择；
- `outcome`：执行结果、失败类型、恢复路径；
- `audit_trace`：状态迁移与决策日志索引。

### 4.3 数据门禁

训练前必须通过：

- schema 完整性检查；
- 关键字段缺失检查（score/risk/cost/failure）；
- 可复现实例抽检；
- 样本去重与时间切分（防止泄漏）。

## 5. 训练流程（四阶段）

### 阶段 A：SFT（结构与约束对齐）

目标：稳定生成合法结构与可执行候选。

要点：

- 先训 Plan，再训 Patch/Replan；
- 强化字段完整性与约束遵循损失。

### 阶段 B：偏好优化（DPO/同类方法）

目标：让模型偏好“成功率高、风险低、改动小”的候选。

偏好对来源：

- 正样本：执行成功、风险可接受、恢复代价低；
- 负样本：执行失败、违反约束、改动过大。

### 阶段 C：失败修复强化

目标：提升失败上下文中的 patch/replan 质量。

要点：

- 引入 evaluator 将失败转为结构化反馈；
- 强化“patch 优先、replan 兜底”的策略学习。

### 阶段 D：蒸馏与部署优化

目标：降低线上时延与成本，同时保持输出质量。

要点：

- 蒸馏/量化；
- 对关键字段与验收指标做回归测试。

## 6. 推理与部署架构（双路保障）

默认路径：

1. 自研 Planner 生成 Top-K 候选；
1. 规则校验器做 schema 与约束检查；
1. 通过后进入自动执行或 HITL 门控。

回退路径（任一条件触发）：

- schema 连续不合法；
- 候选可执行率下降超阈值；
- 连续失败超阈值；
- 风险等级长期偏高。

原则：

- 回退仅替换模型调用层，不改变状态机与决策契约。

## 7. 评估框架与门槛

### 7.1 离线门槛

- Schema 合法率 >= 99.5%
- 可执行 Plan 率 >= 95%
- Patch 最小性命中率 >= 80%
- `suffix_replan` 前缀保持率 = 100%

### 7.2 在线门槛（对比外部基线）

- 首轮执行成功率提升（目标 >= +10%）
- Planner 平均时延下降（目标 >= 30%）
- 人工介入率下降且恢复成功率不下降
- 审计链完整率不下降（PendingAction -> Decision -> EventLog）

## 8. 计划制作与训练落地依赖

后续计划和训练执行必须先落实四项基础资产：

- A1：训练数据规范（字段字典 + 抽取脚本 + 质量门禁）
- A2：候选校验器（schema + 约束 + 风险门控）
- A3：统一 benchmark（离线/在线指标与统计口径）
- A4：发布门禁清单（回归测试 + 回退策略 + 审计对账）

无 A1-A4，不进入正式训练与灰度发布。

______________________________________________________________________

## 9. 风险与控制

- 风险：数据分布偏窄导致泛化不足。
  - 控制：持续采样难例增量训练，保留外部模型回退。
- 风险：模型追求局部成功率但破坏可解释性。
  - 控制：`score_breakdown/explanation` 设为硬性验收字段。
- 风险：训练更新影响现有执行语义。
  - 控制：发布前必须通过 FSM/HITL/恢复链路回归测试。
